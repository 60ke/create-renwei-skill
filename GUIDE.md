# 实操教程：让 ChatGPT 用长期记忆生成专属人味 Skill

## 1. 最简单的方法：直接丢给 ChatGPT

如果你长期使用 ChatGPT，它已经通过记忆和历史对话了解了很多关于你的信息。

最省事的做法不是先整理几十篇文章，而是直接把本仓库的 `PROMPT.md` 交给它，并要求它优先使用：

- 已保存的个人记忆；
- 长期聊天记录；
- 你平时提问和表达判断的方式；
- 你过去对 AI 输出的修改和否定；
- 你长期形成的术语、类比和语气习惯。

可以直接说：

```text
请按照 PROMPT.md 执行。
优先使用你已经保存的关于我的记忆，以及我们过去的长期聊天记录。
不要让我重复整理你已经知道的信息。
参考 renwei-writing 的方法，但不要模仿原作者，只生成属于我的人味写作 Skill。
```

## 2. 让 Agent 阅读参考项目

让 Agent 阅读：

- [orange2ai/renwei-writing](https://github.com/orange2ai/renwei-writing)
- 本仓库的 `PROMPT.md`

参考项目提供的是“保住作者存在感”的方法，不是让 Agent 模仿参考作者。

## 3. 先从记忆和历史聊天中提取风格档案

让 Agent 先填写 `templates/STYLE_PROFILE.md`。

风格档案应该尽量基于它已经掌握的长期信息：

- 哪些习惯在多次对话中出现；
- 哪些表达是你反复使用的；
- 哪些 AI 句式被你多次否定；
- 哪些只属于某个特定场景；
- 哪些只是一次性的情绪或口误。

这一步的重点是把“它记得什么”和“它推断了什么”分开。

如果 Agent 只凭一两句话就宣布“你的风格是直接、犀利、专业”，说明它还没有真正分析。

## 4. 信息不足时，再补最少材料

上传文章不是默认步骤，而是兜底。

只有这些情况需要补材料：

- 新账号，没有足够历史聊天；
- 没有开启记忆；
- Agent 明确看不到旧对话；
- 第一版 Skill 明显不像你；
- 你要单独训练某一种写作身份。

这时再补：

1. 你自己写过并且满意的文章；
2. 未经整理的口述、聊天或语音转写；
3. AI 改稿前后的对照；
4. 你对改稿结果的真实反馈。

不要为了“喂材料”而喂材料。优先补 Agent 当前真正缺少的证据。

## 5. 生成 SKILL.md

让 Agent 按 `templates/SKILL_TEMPLATE.md` 输出 Skill。

一份可用的 Skill 应该包含：

- 作者站在什么位置说话；
- 文章通常怎么起头；
- 观点如何展开；
- 哪些毛边必须保留；
- 哪些词和句式必须避免；
- 修改前如何判断该不该动；
- 修改后如何验收；
- 来自长期对话和真实改稿反馈的正反案例。

## 6. 直接用历史内容做回归测试

第一版生成后，可以直接从过去对话或当前材料中挑一段有代表性的原话，让 Agent 按 Skill 做低干预改写。

重点检查：

- 观点有没有被改软；
- 技术判断有没有被抽象化；
- 合理的犹豫有没有被删掉；
- 有没有增加作者不会使用的文学场景；
- 有没有出现整齐得过头的结构；
- 有没有把自然重复当成错误清理掉；
- 改完之后还能不能认出是同一个人在说话。

## 7. 把失败原因写回 Skill

不要只在对话里说“这次不像我”。

每次反馈都应该变成规则，例如：

```text
反馈：这段太像行业报告。
规则：产品和技术判断优先使用具体功能、成本和执行路径，不替换成“生态、赋能、场景落地”等抽象词。
```

```text
反馈：你把我的判断改得太客观了。
规则：不要为了平衡而主动补反方观点。保留作者明确的个人立场，只在事实错误时纠正。
```

## 8. 维护版本

建议给 Skill 维护简单版本记录：

```text
v0.1：根据 ChatGPT 长期记忆和历史聊天生成初版
v0.2：补充公众号长文结构规则
v0.3：修复过度删除“我觉得”的问题
v0.4：加入技术类比和产品判断案例
```

Skill 不是一次性提示词，更像一个长期维护的个人表达配置。

## 推荐工作流

```text
把 PROMPT.md 丢给 ChatGPT
  ↓
读取长期记忆与历史聊天
  ↓
生成 STYLE_PROFILE.md
  ↓
生成 SKILL.md 与检查清单
  ↓
用历史原话做回归测试
  ↓
信息不足时再补最少材料
  ↓
把失败原因写回 Skill
  ↓
形成稳定版本
```

## 判断是否成功

成功不是文章“更流畅”或者“更高级”。

真正的标准是：

> 文章改顺了，但说话的人没有被换掉。
