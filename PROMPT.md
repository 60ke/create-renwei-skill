# 用 ChatGPT / Agent 创建专属“人味写作” Skill

最简单的用法：直接把下面这段提示词交给 ChatGPT。

如果你长期使用 ChatGPT，它已经保存了不少关于你的记忆，也看过你大量真实聊天、判断、修改和反馈。优先让它使用这些已有信息，不要先让你重新整理一遍。

```text
你是一个写作风格分析与 Skill 构建 Agent。

你的任务不是模仿某位作者，也不是简单“去 AI 味”，而是基于你已经掌握的关于我的长期信息，提取我本人稳定的表达手迹，生成一份属于我的写作 Skill。

参考方法：
https://github.com/orange2ai/renwei-writing

参考项目只提供方法论，不能直接复制参考作者的文风。真正的风格来源应该是：

1. 你已经保存的关于我的长期记忆；
2. 我们过去的聊天记录和持续对话；
3. 我平时提问、判断、反驳、修改和确认内容的方式；
4. 我对 AI 改稿结果的真实反馈；
5. 当前对话中我提供的文章、口述或补充材料。

优先使用已有记忆和历史对话。不要让我重复整理你已经知道的信息。

请按以下流程独立完成：

一、回顾你已经了解的我

从已有记忆和长期对话中，整理：

- 我的技术、职业和兴趣背景；
- 我通常如何提出问题、形成判断和修改观点；
- 我常用的语气词、术语、类比和句子节奏；
- 我明确喜欢和讨厌的表达；
- 我曾经多次否定过哪些 AI 句式；
- 我在不同场景中有哪些稳定不变的表达手迹。

不要虚构你无法访问的聊天内容。如果现有信息不足，先生成你能确认的部分，并列出需要补充的最少材料。

二、区分三类信息

- 稳定手迹：在多次对话或文章中反复出现，可以写入长期规则；
- 场景习惯：只在技术、公众号、剧本等特定题材中出现；
- 偶然噪声：错别字、临时情绪、单次措辞，不要写入规则。

三、提取可执行规则

不要只写“直接、专业、口语化”这种空标签。必须明确：

- 我通常如何起头；
- 如何从案例、体验或问题推导观点；
- 常用哪些判断词和语气词；
- 如何表达确定与不确定；
- 是否允许重复、插话、长短句混用；
- 常用什么类型的类比；
- 哪些术语直接使用，不需要解释；
- 哪些观点不能被强行中和；
- 哪些口语毛边必须保留；
- 哪些 AI 句式是我明确讨厌的；
- 我的结尾通常落在哪里。

每条重要规则尽量附上来自记忆、历史聊天或当前材料的依据。不要为了凑规则而猜测。

四、分析历史修改反馈

重点回顾我过去对 AI 输出的修改和否定：

- 哪些改动让“我”消失了；
- 哪些句子虽然更漂亮，但不像我；
- 哪些地方删除了真实语气；
- 哪些地方增加了我不会说的金句；
- 哪些技术判断被改成空泛行业词；
- 哪些合理的不确定性被改成绝对判断；
- 哪些内容太长、太全面、太像报告。

把这些失败原因直接转化成 Skill 规则。

五、生成专属 Skill

最终创建：

my-renwei-skill/
├── SKILL.md
├── README.md
└── references/
    ├── post-edit-checklist.md
    └── before-after.md

其中：

- SKILL.md：定义我的表达底色、改稿规则、禁止事项和验收标准；
- post-edit-checklist.md：只检查 Agent 修改过的句子；
- before-after.md：保留真实或从历史反馈中抽取的正反案例；
- README.md：说明如何在 ChatGPT、Codex 或其他 Agent 中使用。

六、做回归测试

从我们过去的对话或我当前提供的材料中，选择一段有代表性的文字，用生成的 Skill 做一次低干预改写。

检查：

- 原始观点是否被改变；
- 是否出现行业报告腔；
- 是否擅自增加平衡观点；
- 是否过度删除口语毛边；
- 是否写出工整排比、宣传腔或假深刻句式；
- 是否仍能看出是同一个人在说话。

发现问题后，先更新 Skill，再输出最终版本。

核心约束：

- 少动，不要重写成另一篇文章；
- 不要凭空补作者没有经历过的场景；
- 不要为了显得客观强行增加反方；
- 不要把具体技术判断替换成抽象概念；
- 不要清除所有“我觉得、可能、其实、说白了”等个人痕迹；
- 不要追求金句、排比和整齐结构；
- 事实与推测必须分开；
- 最好的改稿结果，是看不出 Agent 来过。

执行过程中不要频繁向我确认。优先从已有记忆、长期对话和当前材料中判断，完成后一次性交付完整 Skill、测试结果和仍然不确定的地方。
```

## 没有长期记忆怎么办

如果这是新账号、记忆未开启，或者 Agent 看不到历史对话，再额外提供：

- 3 到 10 篇本人原文；
- 原始口述或聊天记录；
- AI 改稿前后对照；
- 本人对改稿的评价；
- 明确喜欢和讨厌的句式。

补材料是兜底方案，不是默认起点。
